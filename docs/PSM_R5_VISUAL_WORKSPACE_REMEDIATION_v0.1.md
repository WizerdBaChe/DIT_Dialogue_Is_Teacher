# DIT R5 視覺工作區修正合約 v0.1

日期：2026-07-19
狀態：D1–D3 已確認並完成實作／自動化預檢；等待使用者最終視覺驗收
目的：修正 R5 視覺驗收中發現的資訊架構問題，不改變資料解析、講解或隱私語意。

## 1. 觀察與不可約痛點

使用者提供的直式畫面顯示，品牌、載入、講解、播放、供應商與語言控制同時佔據頁面上方；主內容必須在相當下方才開始。桌面畫面則同時展示結構側欄、魚骨、全部子代理分支與節點詳情，造成「雖然 DOM 已被清單虛擬化，真正要讀的內容仍被周邊面板壓縮」。

不可約痛點：**使用者一次只想學習或導覽一類資訊，介面卻要求他同時承受所有資訊區塊。**

若不修正，R5 只解決運算與 DOM 成本，沒有解決可視空間與認知負荷；窄螢幕仍不適合實際使用。

最小核心：

- 全域操作在需要時可取用，平時不佔主要閱讀面積。
- 工作區同一時間只顯示一個主要面板。
- 所有面板共用同一個目前節點、播放位置與講解狀態。

本輪不做：可拖曳／可縮放浮動視窗、使用者自訂版面、任意多窗、儲存自訂配置、新的視覺主題。

## 2. 既有設計物件檢查

| 既有物件 | 判定 | 修正 |
|---|---|---|
| `Header` 大型控制列 | 必要功能、錯誤容器 | 拆成常駐精簡列與預設收合的設定匣 |
| `viewMode` 認知／高密度切換 | 與工作區內容重複 | 由明確的工作區分頁取代 |
| `StructurePanel` 桌面常駐側欄 | 導覽有用，但不必常駐 | 改為「結構」工作區分頁 |
| 魚骨＋全部 ribs | 主線有用，全部 ribs 同時展開不必要 | 主線總覽常駐；只虛擬化顯示目前主線節點的 ribs |
| 全部子代理卡片 | 有用，但不應擋在節點詳情上方 | 改為獨立「子代理」分頁與虛擬化摘要清單；選取後回閱讀 |
| 魚骨下方重複節點詳情 | 與閱讀卡片重複 | 移除；從導覽面板選取後切到「閱讀」並定位原卡片 |

## 3. 先驗假設與查證差異

查證前假設：使用「可收合設定列」與「對話／結構／視覺化／詳情」四個分頁；不採浮動視窗。

查證後修正：

- WAI-ARIA Tabs Pattern 定義的正是「一組分層內容一次只顯示一個 panel」，適合主要工作區。
- VS Code 的工作台把樹狀導覽、編輯主區與輔助面板分離，並建議一個容器維持約 3–5 個 Views；同時顯示過多 Views 會造成混亂。
- 「詳情」不應成為第五份重複內容。既有閱讀卡片已是唯一詳情來源；圖解、子代理與結構只負責選取和導覽。
- 浮動視窗會增加遮擋、拖曳、位置保存與鍵盤焦點問題，且在 390 px 寬度沒有可靠空間，因此不納入最小修正。

參考：

- [WAI-ARIA Authoring Practices：Tabs Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/)
- [VS Code UX Guidelines：Sidebars](https://code.visualstudio.com/api/ux-guidelines/sidebars)
- [VS Code User Interface](https://code.visualstudio.com/docs/editing/userinterface)

## 4. 平台無關語意（PIM）

### 4.1 詞彙

- **精簡列（Compact Bar）**：永遠可見，只保留品牌縮寫、工作區切換、上一步／播放／下一步與設定開關。
- **設定匣（Settings Tray）**：預設關閉，集中載入、重設、講解批次、Provider、語言與完整責任說明。
- **工作區（Workspace）**：一次只顯示一個主要學習或導覽內容。
- **目前節點（Active Item）**：跨工作區共用的唯一選取節點。
- **導覽面板（Navigation View）**：魚骨、子代理或結構；負責選取，不複製節點詳情。

### 4.2 不變條件

- **INV-1**：任一時間最多掛載一個主要工作區內容；不是只用 CSS 隱藏。
- **INV-2**：切換工作區不得清除目前節點、播放位置、講解結果或載入文件。
- **INV-3**：從魚骨、子代理或結構手動選取節點後，必須停止播放、清除舊播放位置，並讓閱讀工作區定位到同一 `ViewItem.id`。
- **INV-4**：設定匣關閉時，全域操作仍可由單一可見按鈕重新開啟。
- **INV-5**：載入進度、錯誤與外傳確認不得被設定匣隱藏，因為它們是作業狀態或阻擋性決策。
- **INV-6**：在 390×844 與桌面寬度，精簡列與工作區切換均可用鍵盤操作且焦點可見。
- **INV-7**：50 MiB fixture 在任何工作區的已掛載 DOM 元素維持不超過 250。

### 4.3 工作區狀態

`WorkspaceView = reader | fishbone | subagents | structure`

| 事件 | 原狀態 | 新狀態 | 保留內容 |
|---|---|---|---|
| 點選工作區標籤 | 任意 | 對應工作區 | 目前節點與播放位置 |
| 在導覽面板選取節點 | fishbone / subagents / structure | reader | 新的目前節點 |
| 上一步／下一步／播放 | 任意 | reader | 更新後的播放節點 |
| 開關設定匣 | 任意 | 原工作區 | 全部 session 狀態 |
| 載入成功 | 任意 | reader | 新文件的第一個可讀節點 |

## 5. 語意驗證

### 5.1 需求追溯

| 使用者痛點／需求 | PIM 對應 | 驗收 |
|---|---|---|
| 直式畫面設定占滿上方 | 精簡列、設定匣、INV-4/6 | 390×844 收合狀態上方 chrome 高度不超過 112 px |
| 左側 panel 與欄位壓縮主內容 | 工作區、INV-1 | 同時只存在一個 `tabpanel` |
| 子代理分支卡住詳情 | 導覽面板、INV-3 | 子代理與閱讀不再同頁堆疊 |
| 魚骨可能卡住畫面 | 魚骨按目前 station 虛擬化 ribs、INV-7 | 魚骨工作區 DOM 不超過 250 |
| row list 設定仍需合併 | 設定匣 | 收合後只留一個設定按鈕 |
| 不要畫面混亂 | 四個互斥工作區 | 任何時刻只展示一類主要資訊 |

### 5.2 語意落差登記

| 語意 | Web 平台落差 | 橋接方式 | 是否扭曲 |
|---|---|---|---|
| 一次一個工作區 | 一般 `div` 沒有分頁語意 | `tablist` / `tab` / `tabpanel`、roving tabindex、方向鍵 | 否 |
| 導覽選取後讀取詳情 | 目標卡片可能尚未掛載 | 沿用 ID→虛擬索引橋接，切換後 `scrollToIndex` | 否 |
| 設定暫時展開 | 展開會改變可視高度 | 設定匣限制高度並內部捲動；使用者手動關閉 | 否 |
| 大量 ribs | 全掛載造成 DOM 與視覺密度問題 | 只顯示主線與目前 station 的虛擬 ribs | 主線／rib 關係保留，未扭曲 |

驗證結論：沒有阻擋 PSM 的語意落差；使用者已於 2026-07-19 同意 D1–D3。

## 6. 實作模型（PSM）

### 6.1 推薦版面

收合狀態：

```text
┌ DIT ─ [閱讀] [魚骨] [子代理] [結構] ─ [‹] [播放] [›] [設定] ┐
└─────────────────────────────────────────────────────────────┘
┌──────────────────── 單一工作區內容 ─────────────────────────┐
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

設定匣展開時，在精簡列下方顯示四組內容：Session、Teaching、Provider、Language。直式畫面使用兩列精簡列，工作區標籤可水平捲動；設定匣最高占 `45dvh` 並自行捲動。

### 6.2 檔案與責任

| 檔案 | 變更 |

| `src/components/WorkspaceTabs.tsx` | 新增四分頁、ARIA 與鍵盤行為 |

### 6.4 安全與資料邊界

這次只改本機顯示狀態；沒有新的資料入口、儲存或外傳。既有 Blob streaming、Privacy Gateway 與 secret blocking 路徑不變。設定匣不得隱藏正在等待使用者決定的外傳預覽。
