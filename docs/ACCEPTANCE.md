## 1. 認知模式（魚骨）

- [ ] 側邊欄顯示「蒸餾骨架：主線 4 · 支線 4」。
- [ ] 主線太寬時可**橫向捲動**（此處橫捲為刻意設計）。

## 2. Drill-down（節點點擊展開）
- [ ] 點任一**主線節點** → 下方「節點詳情」展開該步的完整卡片（含思考/操作/結果）。
- [ ] 點任一**支線** → 詳情切換到對應內容（例：點 ⚠ 錯誤 → 顯示 `Bash npm test` 與失敗輸出）。
- [ ] 被選取的節點/支線有**高亮**狀態。

## 3. 高密度模式
- [ ] 右上切「高密度」→ 改為**逐步卡片時間軸**（約 16 張）。
- [ ] 卡片含徽章（里程碑/決策點/重試/錯誤）、可展開**思考鏈**、可摺疊**參數/結果**區塊。

## 4. 重播
- [ ] 按 ▶ 重播 → 逐步高亮節點；按鈕變「⏸ 暫停」；⏮ ⏭ 可上一步/下一步。

## 5. 講解來源 / 隱私（責任釐清）
- [ ] 切換「講解來源」下拉，**頂部說明橫幅即時變色變字**：
  - 不講解 → 「零外傳」（綠灰）
  - 本地 Ollama → 「不外傳」（teal）
  - 雲端 API → 「將傳送至外部服務…責任歸使用者」（琥珀）
- [ ] 選「雲端」會出現「產生教學講解」按鈕；點下去顯示「尚未設定」訊息（樁，正常）。
- [ ] 選「本地 Ollama」→ 下方出現**引導面板**，即時顯示連線狀態：
  - 未啟動/被 CORS 擋 → 「無法連線」+ 可複製的 `ollama serve`（含 `OLLAMA_ORIGINS`）指令。
  - 已啟動但**沒有模型** → 「已連線・尚無模型」+ 可複製的 `ollama pull <model>`（修正先前 404 痛點）。
  - 已安裝其他模型 → 可用**模型下拉**直接切換；按「重新檢查」重新探測。
- [ ]（選用，需本機 Ollama + 已 pull 模型）面板顯示「已就緒」→「講解全部」→ 各節點出現 what/why/通用做法；
      模型不存在時 annotate 會回傳「請 `ollama pull <model>`」訊息，不應整頁崩潰。
