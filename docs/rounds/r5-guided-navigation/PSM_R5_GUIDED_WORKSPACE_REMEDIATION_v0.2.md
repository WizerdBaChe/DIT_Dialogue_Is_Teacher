# DIT R5 引導式工作區修正合約 v0.2

日期：2026-07-19
狀態：設計完成，等待 D4–D6 使用者確認；尚未實作
取代範圍：取代 `PSM_R5_VISUAL_WORKSPACE_REMEDIATION_v0.1.md` 的工作區資訊架構與 D2；保留設定匣、虛擬化、串流匯入、共用選取與效能成果
目的：恢復 DIT 原始的「左側結構＋右側學習內容」，並加入可重返、可操作的首次與每次 Session 引導。

## 0. 修正結論

v0.1 把「結構」誤判為可與閱讀互斥的內容頁，造成兩個概念錯誤：

1. 使用者失去「目前在哪裡、接下來去哪裡」的持續方位感。
2. 預設直接進入閱讀，使已降噪的內容仍成為沒有任務脈絡的一長串卡片。

本版採用以下唯一推薦方向：

- **結構是導航骨架**：在可容納雙欄的寬度固定於左側；不是工作區分頁。
- **總覽是可重返的使用指南**：啟動、載入成功與重設後先進總覽，不直接進閱讀。
- **閱讀是主要任務**：開始／繼續、上一步、重播、下一步都回到閱讀並同步左側位置。
- **魚骨與子代理是延伸視角**：使用者理解主線後再主動進入，不排在必要流程前。
- **設定與講解是選配**：保留預設收合設定匣；不阻擋基本複盤。

## 1. 第一性原理（CIM）

### 1.1 真正任務

使用者不是來「閱讀一份紀錄」，而是要在一次代理任務完成後，快速回答：

1. 這次任務要解決什麼？
2. 代理採取了哪些關鍵步驟，我目前看到哪一步？
3. 為什麼這樣做，有哪些可重用的方法？
4. 哪些分支、失敗或委派值得再深入？

### 1.2 不可約痛點

如果畫面只呈現內容，不呈現任務順序與位置，使用者仍需自行從大量資訊中推導「先看什麼、現在在哪裡、下一步是什麼」。這正是 DIT 應消除的工作。

### 1.3 成功條件

- 第一次進入者在 10 秒內能指出主要起點。
- 載入 Session 後，使用者先取得任務與資料概況，再決定開始或繼續閱讀。
- 閱讀、魚骨與子代理之間切換時，左側持續顯示同一目前位置。
- 使用者可在 1 分鐘內藉左側結構與總覽說出 Session 的關鍵步驟骨架；沿用 RPD 原始驗證目標。

### 1.4 不做

- 不新增學習完成度、書籤、筆記或跨 Session 進度同步。
- 不加入可拖曳／浮動多窗、任意版面配置或 Activity Bar 仿製品。
- 不用一次性泡泡導覽取代永久可回訪的說明。
- 不改變資料解析、降噪、講解、隱私與外傳同意語意。

## 2. 正確使用順序的查證

### 2.1 DIT 自身契約

原始 RPD 已定義使用情境為「任務結束後複盤」，典型流程是：

`選擇 Session → 解析／降噪 → [可選] 講解 → 左側樹＋右側內容 → 瀏覽／展開／重播`

產品價值順序也明確是：先「一眼看懂關鍵步驟骨架」，再看逐步 why、通用做法與重播。結論是：**結構與閱讀必須同時成立，講解與延伸視圖不得成為入口前置作業。**

內部依據：`RPD_DIT_v0.1.md` §C、§E、§G、§K。

## 3. 使用者旅程

### 3.1 首次進入（內建範例）

1. 系統載入內建範例，但主內容顯示「總覽」，不是閱讀卡片。
2. 總覽明示「目前是內建示範 Session」，以一句話說明 DIT 的用途。
3. 首屏顯示三步：
   - **確認 Session**：看任務名稱、來源、節點數與解析警告。
   - **沿主線閱讀**：左側看位置，右側逐步閱讀或重播。
   - **延伸理解**：需要時再看 why、魚骨與子代理。
4. 主按鈕「開始示範」進閱讀第一項；次按鈕「載入自己的 Session」開啟既有檔案選擇。

### 3.2 載入自己的 Session

1. 使用者從設定匣載入 `.jsonl` 或 Session 資料夾。
2. 載入進度與錯誤保持全域可見。
3. 成功後進總覽，顯示新 Session 摘要、解析警告與本機處理說明。
4. 主按鈕「開始閱讀」進第一項；若 Session 已有目前位置則顯示「繼續閱讀」。
5. 左側結構同步高亮目前項目，可直接跳到任意節點。

### 3.3 複盤中的日常流程

`總覽 → 閱讀主線 ↔ 左側結構跳轉 → 展開節點 why → 魚骨總結 → 子代理分支 → 回閱讀`

魚骨與子代理不是完成閱讀的強制關卡；它們是使用者有疑問時的延伸視角。

## 4. 平台無關模型（PIM）

### 4.1 詞彙

- **總覽（Overview）**：Session 導向頁，也是永久使用指南。
- **主要視角（Primary View）**：總覽、閱讀、魚骨、子代理四者之一。
- **結構側欄（Structure Sidebar）**：目前 Session 的虛擬化樹狀導航；不屬於主要視角。
- **目前位置（Current Position）**：目前選取節點在可讀項目中的索引。這不是「已學會」或「已完成」進度。
- **延伸視角（Secondary Perspective）**：魚骨與子代理；共享目前位置但不複製完整節點詳情。

### 4.2 不變條件

- **INV-1**：寬度足以雙欄時，結構側欄與任何主要視角並存。
- **INV-2**：結構側欄永遠只承載 Session 名稱、目前位置、收合控制與虛擬化樹；Session 詳細 metadata、圖例與設定移到總覽／設定匣。
- **INV-3**：切換主要視角不得清除文件、目前位置或講解結果。
- **INV-4**：任何手動視角切換或結構跳轉都暫停重播，避免位置在不可見視角中自行移動。
- **INV-5**：上一步、重播、下一步一律切回閱讀，並同步左側 active row。
- **INV-6**：載入成功與重設後進總覽；不得直接顯示第一坨閱讀內容。
- **INV-7**：總覽每個引導步驟都必須有可執行的動詞或明確去向，不只描述功能。
- **INV-8**：窄螢幕收合結構時，必須保留「結構」觸發按鈕與目前位置摘要；不得讓導航消失。
- **INV-9**：50 MiB fixture 同時掛載側欄與主要視角時，DOM elements 不超過 250；兩份清單均保留虛擬化。
- **INV-10**：所有阻擋性錯誤、載入狀態與 Privacy Review 不得藏在設定匣或被總覽取代。

### 4.3 狀態機

`PrimaryView = overview | reader | fishbone | subagents`

| 事件 | 新視角 | 目前位置 | 重播 |
|---|---|---|---|
| App 啟動、內建範例載入完成 | overview | 第一個可讀項目 | idle |
| 使用者 Session 載入完成 | overview | 第一個可讀項目 | idle |
| 重設 | overview | 第一個可讀項目 | idle |
| 開始／繼續閱讀 | reader | 保留目前位置 | idle |
| 左側結構選取 | reader | 改為所選項目 | paused |
| 切換總覽／魚骨／子代理 | 對應視角 | 保留 | paused |
| 上一步／下一步／重播 | reader | 依操作更新 | 對應播放狀態 |
| 魚骨／子代理選取節點 | reader | 改為所選項目 | paused |

## 6. 語意驗證

### 6.1 需求追溯

| 使用者問題 | 設計回應 | 驗收證據 |
|---|---|---|
| 進入後不知道要做什麼 | 總覽預設頁、三步引導、兩個動詞 CTA | 首屏 10 秒指向測試；App 啟動不掛載 Reader |
| 沒有使用手冊 | 總覽即永久指南，可隨時返回 | 導航永遠有「總覽」；不依賴只出現一次的提示 |
| 沒有正確順序 | 明定載入→總覽→閱讀→延伸 | 狀態轉移測試＋UAT 操作腳本 |
| 結構應留左側看位置與跳轉 | Structure Sidebar 與主要視角並存 | 740／1280／2048 寬度可見且 active row 同步 |
| 窄畫面不能再被 panel 壓縮 | <720 px 改左側抽屜，仍保留位置觸發 | 390×844 無水平溢出；抽屜可鍵盤關閉 |
| 大 Session 不可回到大量 DOM | Sidebar 與主要內容各自虛擬化 | 50 MiB 每視角 DOM ≤250 |

### 6.2 語意落差登記

| 語意 | Web 實作落差 | 橋接 | 結論 |
|---|---|---|---|
| 結構持續存在 | 390 px 無法同時給側欄與可讀主區 | 左側 modal drawer＋可見位置摘要 | 保留導航語意，不扭曲 |
| 「進度」 | 目前只有索引，沒有學習完成紀錄 | UI 使用「位置」而非「完成進度」 | 避免錯誤承諾 |
| 列表節點可能未掛載 | 虛擬化只渲染視窗附近 | ID→index→scrollToIndex，完成後聚焦／高亮 | 可實現 |
| 總覽與 Reader 不同掛載 | 進總覽可能讓重播在背景繼續 | 手動切換一律 pause | 避免隱形狀態變化 |

結論：沒有阻擋實作的語意落差；但 <720 px 的抽屜行為、每次載入是否進總覽，以及是否拒絕一次性導覽套件屬 UX 語意，需使用者確認。

### 7.3 失敗與降級可見性

- 範例或 Session 載入失敗：總覽顯示可讀錯誤與「重新載入／載入其他 Session」，不得空白。
- Session 無魚骨或子代理：對應分頁顯示原因與回閱讀 CTA，不顯示空 panel。
- 抽屜焦點管理失敗的自動化測試不得略過；這是窄螢幕主要導航。
- 虛擬定位找不到 ID：保留目前可見內容並顯示結構化診斷，不靜默跳到第一項。

### 7.4 安全與資料邊界

本版只改本機 UI 狀態與文案；不新增外傳、儲存或資料解析。Privacy Gateway、Secret blocking、Blob streaming、Worker cancellation 與講解快取均維持原契約。

## 8. 驗收與回歸

### 8.1 自動化

- 狀態：啟動／載入／重設→overview；結構選取→reader＋paused；播放控制→reader。
- 結構：桌面與 Reader 同時掛載；手動收合後仍有可用 trigger。
- 窄螢幕：drawer role、focus trap、Escape、焦點回復、選取後關閉。
- i18n：zh-TW／EN 所有新 key 齊全，CTA 不退回原始 key。
- 效能：50 MiB fixture 在四個主要視角逐一量測 DOM ≤250。
- 全量：既有 95 tests、typecheck、production build、`git diff --check`。

### 8.2 實機 UAT

1. **首次入口**：390×844、740×1113、1280×720 進入時均看到「從這裡開始」與主要 CTA，而非閱讀卡片。
2. **桌面方位**：740／1280／2048 寬度左側結構預設可見；切總覽、閱讀、魚骨、子代理都不消失。
3. **主線開始**：按「開始示範／開始閱讀」後進 Reader；左側第一項 active。
4. **同步定位**：下一步、上一項、重播與深層結構跳轉都讓左側與右側指向同一項。
5. **窄螢幕導航**：390 px 按「結構」由左側開抽屜；選節點後抽屜關閉並定位閱讀。
6. **可重返指南**：任何時候按「總覽」可回看順序；目前位置不被清除。
7. **載入新 Session**：成功後先看新摘要，不直接掉進第一張卡。
8. **資訊密度**：設定匣預設收合；左側不再重複完整 metadata 與圖例。
9. **延伸視角**：無魚骨／子代理時顯示明確空狀態；有資料時選取可回 Reader。
10. **雙語與鍵盤**：中英文排序一致；tab、sidebar、drawer 與 CTA 焦點可見。
