# DIT R5 遊戲式導航概念評估 v0.1

日期：2026-07-19
狀態：推薦採用，等待使用者確認互動語意；不是施工合約
關聯文件：`PSM_R5_GUIDED_WORKSPACE_REMEDIATION_v0.2.md`

## 1. 結論

將魚骨改造成「右下微縮導航＋可開啟的大型地圖」能提高 DIT 可用性，前提是它與左側結構分工，而不是重複：

- 左側結構回答「這一步叫什麼、父子關係是什麼、我要精確跳去哪裡」。
- 魚骨地圖回答「整段 Session 長什麼樣、我在全局哪裡、重要地標與支線分布在哪裡」。
- 中央閱讀回答「這一步做了什麼、為什麼、結果如何」。

這是空間導航模式，不是把 DIT 遊戲化。推薦借用遊戲的定位、地標、快捷地圖與情境提示；拒絕 XP、連續登入、排行榜與假完成度。

## 2. 不可約痛點與核心能力

不可約痛點：長 Session 中，使用者即使能在樹上找到節點，仍難以感知整體任務形狀、目前位置與重要分岔。

最小核心：

1. 持續顯示「你在這裡」。
2. 用少量穩定符號顯示全局主線與重要地標。
3. 一個動作打開可探索地圖。
4. 從地圖選取節點後，可靠回到同一 Reader 項目。

不做：遊戲美術 HUD、動畫獎勵、分數、角色成長、強迫探索、隱藏既有資料的迷霧。

## 3. 查證結果

- VS Code 官方將 Minimap 定義為原始內容的高階總覽，用於快速導航與理解；可選取或拖曳視窗區跳轉。DIT 可採用「總覽＋目前視窗」概念，但不能逐節點等比例縮小。
- React Flow 的 MiniMap 能顯示節點、目前 viewport，並支援節點點擊、平移與縮放；功能適合圖編輯器，但完整套件對目前自製魚骨偏重。
- Xbox Accessibility Guidelines 要求地圖放大後仍提供不依賴捲動的替代導航，例如文字化地標清單。DIT 已有左側結構，正好成為地圖的等價替代路徑。
- W3C 對單字元快捷鍵要求可關閉、可重設，或只在元件聚焦時啟用；因此 `M` 不得成為唯一入口，也不得在文字輸入中攔截。
- W3C 建議指標目標至少 24×24 CSS px；微縮地圖中的數千小點不適合逐點直接點擊。

## 4. 推薦互動

### 4.1 右下微縮導航

- 只在 Reader 顯示，預設為安靜的 176×112 px 區塊；740 px 寬度縮為 144×96 px。
- 顯示：主線、里程碑、決策、錯誤、子代理分岔、目前位置與目前閱讀視窗。
- 不顯示節點名稱、參數、結果或全部 ribs。
- 整張微縮圖是一個可操作目標：點擊、`Enter` 或 `M` 開啟大型地圖。
- 不讓使用者在微縮圖直接點擊密集小點，避免誤跳與無法滿足命中尺寸。
- 可由設定關閉；關閉後仍保留「開啟地圖」按鈕。

### 4.2 大型魚骨地圖

- 桌面為置中的 modal：最大 88vw × 84dvh，背景降噪但仍保留工作區輪廓。
- 720–899 px 為 92vw × 88dvh；小於 720 px 改全螢幕 sheet。
- `M` 開啟／關閉，`Escape` 關閉，另有永遠可見的地圖與關閉按鈕。
- 開啟時以目前節點置中，顯示「你在這裡」。
- 遠距縮放只顯示里程碑與分支叢集；中距顯示 spine；近距才顯示 ribs 與文字標籤。
- 單擊地標只選取並顯示一行摘要；`Enter`、雙擊或「跳到這一步」才真正跳轉，降低誤觸。
- 跳轉後關閉地圖、停止重播、切回 Reader、同步左側 active row 並聚焦目標卡片。
- 另提供與可見地標同步的文字清單；鍵盤與輔助科技不需操作二維畫布。

### 4.3 快捷鍵規則

- 預設 `M`，但在 `input`、`textarea`、`select`、`contenteditable` 或任何設定欄位聚焦時無效。
- 設定提供「關閉單鍵快捷鍵」；未來若加入鍵盤設定，可改為含修飾鍵組合。
- 快捷鍵只加速既有按鈕，不承載獨占功能。

## 5. 大型 Session 的語意縮放

29,452 個項目不能變成 29,452 個 DOM／SVG hit targets。地圖模型需先產生固定上限的視覺層級：

| 縮放層級 | 顯示內容 | 上限方向 |
|---|---|---|
| L0 全局 | 里程碑、主要分支叢集、目前位置 | 約 80 個地標 |
| L1 區段 | 目前里程碑附近 spine、決策、錯誤、子代理入口 | 約 200 個 glyph |
| L2 細節 | viewport 周圍 ribs 與標籤 | 虛擬化、約 120 個可互動項目 |

聚合只改變畫面表示，不改變 `ViewItem.id`、spine/rib 關係或跳轉目標。全局層的叢集不可宣稱是單一真實節點。

## 6. 可借用的其他遊戲設計

| 遊戲模式 | DIT 對應 | 推薦度 | 邊界 |
|---|---|---:|---|
| Mission Brief／任務簡報 | 總覽先說明任務、Session 與三步用法 | 高 | 使用一般產品文案，不硬叫「任務簡報」 |
| Quest Tracker／當前目標 | Reader 頂部顯示目前里程碑、目前步驟與下一個合理動作 | 高 | 稱「目前目標／位置」，不稱完成度 |
| World Map＋Minimap | 魚骨微縮導航與大型地圖 | 高 | 不取代文字結構 |
| Landmark／地標 | 里程碑、決策、錯誤、結果、子代理入口 | 高 | 符號需沿用 DIT 既有語意 |
| Fast Travel／快速移動 | 從結構或大型地圖跳到 Reader 節點 | 高 | 跳轉後必須可回溯且停止舊播放 |
| Pause Menu／暫停選單 | 預設收合設定匣 | 已採用 | 不把必要操作藏入設定 |
| Contextual Hint／情境提示 | 首次遇到魚骨、講解或子代理時提供一次短提示 | 中 | 必須可略過且在總覽可重看 |
| Codex／圖鑑 | 未來彙整可帶走觀念、術語與常見模式 | 中／未來 | 屬新知識資產，不能混入 R5 |
| Checkpoint／檢查點 | 里程碑快速清單與回到上一里程碑 | 中 | 不新增假的保存／完成語意 |
| Fog of War／戰爭迷霧 | 隱藏未讀內容 | 低，拒絕 | 會阻礙複盤與全局理解 |
| XP、徽章、連續登入 | 學習獎勵 | 低，拒絕 | 容易把理解變成刷數字 |

## 7. 技術選型盤點

2026-07-19 npm registry：

| 候選 | 版本 | 授權 | 解壓大小 | 判定 |
|---|---:|---|---:|---|
| `@xyflow/react` | 12.11.2 | MIT | 1,208,222 B | 功能完整但會要求魚骨遷入其 node/edge/viewport 模型，整合面過大 |
| `d3-zoom` | 3.0.0 | ISC | 86,878 B | 小而成熟，但 npm 最後更新 2022；只處理 zoom transform，鍵盤與語意仍需自建 |
| `react-zoom-pan-pinch` | 4.0.3 | MIT | 664,402 B | 近期維護、可包現有 SVG；體積與抽象仍超過第一版所需 |
| 現有 SVG＋原生 pointer／wheel | 既有 | 0 新依賴 | 0 | **推薦第一版**：魚骨已是 SVG，先做固定 zoom levels、fit/current 與平移，不做通用圖編輯器 |

若確認實作，建議先沿用現有 SVG；若 UAT 證明平移／縮放手勢難以可靠維護，再由使用者決定是否引入 `react-zoom-pan-pinch`。不推薦為 MiniMap 單一功能遷入 React Flow。

## 8. 與左側結構及 v0.2 的關係

- v0.2 的左側結構不變，仍是持續方位與精確文字跳轉。
- 若本概念獲准，「魚骨」從主要視角分頁移除，改由 Reader 右下微縮圖與 Header 的「地圖」命令開啟。
- 總覽仍可提供「查看全局地圖」次要 CTA，但不在首次流程中強迫開圖。
- 子代理仍保留獨立視角；大型地圖只顯示其入口與分岔，不承載完整分支內容。
